# An Agent Action Is a Hypothesis Until Admitted

[← Back to Semantic Admission Index](README.md)

## Core Claim

A human idea is not a fact just because it appears in someone’s mind.

It becomes knowledge only after validation.

We test it. We compare it against reality. We look for contradictions. We ask whether the reasoning actually holds.

Engineering works the same way.

An intuition is not a system invariant.

A design idea is not a production guarantee.

A theory is not accepted truth until it survives contact with evidence.

So why should an AI agent be treated differently?

## Model Output Is Not Truth

A model output is not system truth.

A proposed tool call is a candidate.

A proposed event is a candidate.

A proposed context update is a candidate.

A proposed state-changing action is a candidate.

Until the system admits it, it should not become durable fact.

## The Admission Boundary

The principle behind an admission boundary is simple:

> An agent action is a hypothesis until admitted.

The boundary does not ask only:

> Did the agent execute?

It asks:

> Should this candidate action be allowed to become accepted history?

That distinction matters.

Without it, a system may allow probabilistic outputs to mutate enterprise state as if they were validated facts.

That is not governance.

That is architectural over-trust.

## Evaluation, Observability, and Admission

Evaluation can improve the agent.

Observability can show what happened.

Admission decides what is allowed to become system truth.

For state-changing agent systems, all three matter.

But only admission protects the moment where a generated action becomes durable reality.
